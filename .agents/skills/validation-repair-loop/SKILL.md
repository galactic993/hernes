# 検証修復ループスキル

## 目的

検証フィードバックを使って、動く実装へ収束させる。

## 入力

- 失敗したコマンド出力
- `specs/<feature>/acceptance.md`
- `specs/<feature>/evidence.md`
- `specs/<feature>/loop-log.md`
- 現在の変更ファイル

## 出力

- 修復したコード
- `loop-log.md` の更新
- `evidence.md` の更新

## 手順

1. 失敗出力を読む。
2. 根本原因を特定する。
3. 失敗の種類を述べる：実装バグ / テストバグ / 仕様の曖昧さ / 環境問題。
4. 実装バグなら、最小の修正を当てる。
5. テストバグなら、仕様と明確に矛盾する場合のみテストを直す。
6. 仕様の曖昧さなら、停止してハンドオフを書く。
7. 最も狭い失敗テストを最初に実行する。
8. 通ったら `make verify` を実行する。
9. 試行を記録する。

## 停止条件（必須）

次の場合は停止し、人間へのハンドオフを書く：
- 同じ失敗が2回繰り返す
- 修復試行が3回失敗
- 修復にプロダクト判断が必要
- 修復に新規依存が必要
- 修復が無関係な挙動を壊すおそれ
